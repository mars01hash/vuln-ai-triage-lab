# Vuln AI Triage Lab - v2 থেকে v3 এর আপডেট সমূহ

এই ডকুমেন্টে Vuln AI Triage Lab এর দ্বিতীয় সংস্করণ (v2) থেকে তৃতীয় সংস্করণ (v3) এ কী কী পরিবর্তন ও নতুন ফিচার যুক্ত করা হয়েছে তার বিস্তারিত বিবরণ দেওয়া হলো।

---

## ১. সংক্ষেপে v2 এবং v3 এর তুলনামূলক পার্থক্য

| ফিচার এলাকা | সংস্করণ ২ (v2) | সংস্করণ ৩ (v3) |
| :--- | :--- | :--- |
| **এমবেডিং মডিউল** | শুধুমাত্র কাস্টম `HashEmbedding` ক্লাস ছিল। | **অ্যাবস্ট্রাক্ট এমবেডিং প্রোভাইডার**। লোকাল হ্যাশিং বা Hugging Face এর রিয়াল নিউরাল এমবেডিং মডেল ব্যবহারযোগ্য। |
| **মেমরি ব্যাকএন্ড** | সাধারণ JSON ফাইলে রেকর্ড সেভ হতো। | **SQLite ও ChromaDB** ব্যাকএন্ড সমর্থন করে। ডেটা ডিস্কে রিলেশনাল ডাটাবেজ আকারে সংরক্ষিত হয়। |
| **ট্রিয়েজ এজেন্ট** | স্ট্যাটিক পাইথন রুল দিয়ে ট্রিয়েজ করা হতো। | **ঐচ্ছিক OpenAI LLM এজেন্ট** (JSON মোডে) এবং এরর ফলব্যাক মেকানিজম যুক্ত করা হয়েছে। |
| **অনুমোদন পলিসি** | কোনো নির্দিষ্ট অনুমোদন গেটিং কোড ছিল না। | **ডিটারমিনিস্টিক অনুমোদন পলিসি** (`approval_policy.py`) যুক্ত করা হয়েছে যা এপ্রুভাল কিউ ট্র্যাক করে। |
| **হিউম্যান ফিডব্যাক** | হিউম্যান রিভিউ ট্র্যাক করার কোনো সিস্টেম ছিল না। | **ফিডব্যাক স্টোর** (`api_human_feedback.jsonl`) এবং ফিডব্যাক অ্যান্ডপয়েন্ট যুক্ত করা হয়েছে। |
| **ডকার সাপোর্ট** | ডকার কন্টেইনারাইজেশন ছিল না। | এপিআই কন্টেইনার রান করতে **Docker Compose** সমর্থন যুক্ত করা হয়েছে। |

## ২. নতুন মডিউল ও কোড ফাইলের বিস্তারিত বিবরণ

সংস্করণ ৩ (v3) এ যুক্ত হওয়া নতুন ফাইলগুলোর ভূমিকা নিচে ব্যাখ্যা করা হলো:

### ক. এমবেডিং প্রোভাইডার অ্যাবস্ট্রাকশন
* **ফাইল:** [providers.py](file:///g:/vuln-ai-triage-lab/app/embeddings/providers.py)
* **কাজ:** এমবেডিং তৈরি প্রক্রিয়াকে আরও ফ্লেক্সিবল করতে `EmbeddingProvider` ইন্টারফেস যোগ করা হয়েছে।
  * `HashEmbeddingProvider`: ২৫৬-ডাইমেনশনের লোকাল SHA256 সাইন হ্যাশিং ব্যাকএন্ড।
  * `SentenceTransformerEmbeddingProvider`: আসল নিউরাল টেক্সট এমবেডিং মডেল `all-MiniLM-L6-v2` লোড ও রান করতে পারে (ডিপেন্ডেন্সি: `sentence-transformers` লাইব্রেরি)।

### খ. SQLite ও Chroma ভেক্টর মেমরি
* **ফাইল:**
  * [sqlite_vector_memory.py](file:///g:/vuln-ai-triage-lab/app/storage/sqlite_vector_memory.py)
  * [chroma_memory_store.py](file:///g:/vuln-ai-triage-lab/app/storage/chroma_memory_store.py)
* **কাজ:** মেমরি স্টোরেজ ব্যাকএন্ড ভেক্টর ডাটাবেজে উন্নীত করা হয়েছে।
  * SQLite ব্যাকএন্ডটি ভেক্টর অ্যারেগুলোকে জেসন স্ট্রিং হিসেবে রিলেশনাল ডাটাবেজে সেভ করে এবং পাইথনে কোসাইন সিমিলারিটি ক্যালকুলেট করে।
  * ChromaDB আসল এন্টারপ্রাইজ গ্রেড ভেক্টর ডাটাবেজের মতো সরাসরি কন্টেন্ট কুয়েরি ও ডুপ্লিকেট মার্জ সম্পন্ন করে।

### গ. ঐচ্ছিক OpenAI LLM এজেন্ট
* **ফাইল:** [llm_agent.py](file:///g:/vuln-ai-triage-lab/app/agents/llm_agent.py)
* **কাজ:** OpenAI এপিআই ব্যবহার করে ত্রুটির সামারি ও ফিক্স গাইড জেনারেট করা।
* **কোড লজিক:**
  * এটি OpenAI JSON Mode এবং `gpt-4o-mini` মডেল ব্যবহার করে ফাইন্ডিং রিভিউ করে।
  * কোনো কারণে এপিআই কল ব্যর্থ হলে (যেমন: API key না থাকা, লিমিট শেষ হওয়া, নেটওয়ার্ক এরর) এটি স্বয়ংক্রিয়ভাবে লোকাল কন্ডিশনাল এজেন্টে ফলব্যাক করে এবং এরর মেসেজটি সামারিতে লিখে দেয়।

### ঘ. ডিটারমিনিস্টিক অনুমোদন পলিসি (Approval Policy)
* **ফাইল:** [approval_policy.py](file:///g:/vuln-ai-triage-lab/app/policy/approval_policy.py)
* **কাজ:** কোড লেভেলে মানুষের রিভিউ ও অনুমোদন নিশ্চিত করা।
* **কোড লজিক:**
  * ফাইন্ডিংয়ের আউটপুট চেক করে যদি WAF প্রস্তাবনা থাকে, ঝুঁকি স্তর critical বা high হয়, অথবা ডুপ্লিকেট গ্রুপ ধরা পড়ে, তবে সংশ্লিষ্ট অ্যাকশনটি `approval_required_actions` ফিল্ডে পুশ করে দেয়।

### ঙ. হিউম্যান ফিডব্যাক স্টোর
* **ফাইল:** [feedback_store.py](file:///g:/vuln-ai-triage-lab/app/feedback/feedback_store.py)
* **কাজ:** মানুষের দেওয়া মতামত অ্যাপেন্ড-অনলি জেসন ফাইল `api_human_feedback.jsonl` এ রাইট করা।

## ৩. ইন্টিগ্রেশন ও রানিং কমান্ডের পরিবর্তন

v3 পাইপলাইনে বেশ কিছু নতুন CLI ফ্ল্যাগ ও API অ্যান্ডপয়েন্ট যুক্ত করা হয়েছে:

### ক. CLI কমান্ডে নতুন ফ্ল্যাগ সমূহ
```bash
python -m app.cli --input data/sample_findings_all.json --use-ml --memory-backend sqlite --memory-file output/v3_memory.sqlite --use-llm --llm-model gpt-4o-mini --report output/v3_report.md --pretty
```
* `--memory-backend`: মেমরি সংরক্ষণ করার ব্যাকএন্ড টাইপ নির্ধারণ করে (`json` অথবা `sqlite`)।
* `--use-llm`: ঐচ্ছিক OpenAI LLM ট্রিয়েজ এজেন্ট সক্রিয় করে।
* `--llm-model`: এপিআই কুয়েরির জন্য OpenAI মডেল সিলেক্ট করে (ডিফল্ট: `gpt-4o-mini`)।

### খ. FastAPI এপিআই পরিবর্তন
* **নতুন Query Parameter:** POST `/triage?use_ml=true&use_llm=true` এবং POST `/triage/batch?use_ml=true&use_llm=true` পাঠিয়ে এপিআইতে সরাসরি ML এবং LLM উভয় মোড সক্রিয় করা যায়।
* **নতুন POST অ্যান্ডপয়েন্ট:** `/feedback` (মানুষের রিভিউ এপ্রুভাল এপিআইতে সেন্ড করতে)।
* **নতুন GET অ্যান্ডপয়েন্ট:** `/feedback/summary` (জমা হওয়া ফিডব্যাক সমূহের পরিসংখ্যান দেখতে)।

## ৪. কেন এই আপডেটগুলো করা হলো? (Rationale)

১. **প্রোডাকশন-রেডি আর্কিটেকচার:** ফ্ল্যাট জেসন ফাইলের পরিবর্তে রিলেশনাল ডাটাবেজ (SQLite) এবং ভেক্টর ডাটাবেজ (ChromaDB) ইন্টিগ্রেশনের মাধ্যমে বড় স্কেলে প্রজেক্ট রান করার উপযোগী করা হয়েছে।
২. **অ্যাডভাইজরি বনাম কন্ট্রোল সেপারেশন:** WAF এবং প্রায়োরিটি স্কোরিংয়ের মতো ক্রিটিক্যাল সিদ্ধান্তগুলো কোডে ডিটারমিনিস্টিক রাখা হয়েছে, অথচ LLM দিয়ে চমৎকার সামারি ও গাইড প্রস্তুত করার সুবিধা নেওয়া হয়েছে। এটি এআই জেনারেটেড কোডের নিরাপত্তা বহুগুণ বাড়িয়ে দেয়।
৩. **মডেলের দীর্ঘমেয়াদী উন্নতি:** হিউম্যান ফিডব্যাক স্টোরের মাধ্যমে মানুষের দেওয়া ভুল-ত্রুটি সংশোধনের ট্র্যাকিং ডেটা সংরক্ষণ করে পরবর্তী সময়ে মডেল ফাইন-টিউনিং ও কোয়ালিটি নিশ্চিত করা সম্ভব।
